## 2024-05-23 - [Critical] Path Traversal Prefix Bypass in Storage Router
**Vulnerability:** The `lib/storage/storageRouter.ts` used `!resolvedTarget.startsWith(publicDir)` to prevent path traversal when handling file uploads. This is vulnerable to a prefix bypass attack. If `publicDir` is `/app/public`, and a user requests a path that resolves to `/app/public-secret` (e.g. by passing `../public-secret`), the `startsWith` check returns `true`, allowing the file upload to bypass the intended public directory boundary.
**Learning:** Checking `path.startsWith(prefix)` in Node.js paths is insufficient for security boundaries because it checks raw strings, not directory segments. A sibling directory that shares the same name prefix as the target directory will incorrectly pass the validation.
**Prevention:** Always enforce directory boundaries by appending the path separator (`path.sep`) to the prefix check, and explicitly checking for an exact match to the root directory itself if files directly within the root are allowed. The correct pattern is: `!resolvedTarget.startsWith(publicDir + path.sep) && resolvedTarget !== publicDir`.
