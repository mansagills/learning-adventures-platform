## 2025-02-28 - Insecure Randomness in Certificate Verification Codes
**Vulnerability:** The `generateVerificationCode` function in `lib/certificates/certificateUtils.ts` used `Math.random()` to generate characters for a 12-character verification code.
**Learning:** Certificate verification codes are used as sensitive tokens to prove authenticity. Generating them using `Math.random()` allows an attacker to potentially predict or brute-force valid verification codes because `Math.random()` is not cryptographically secure and seed states can be guessed.
**Prevention:** Always use Node.js's built-in `crypto` module (e.g., `crypto.randomBytes()`) when generating passwords, secrets, verification codes, or any other token meant to provide security or authenticity, rather than relying on pseudo-random generators.
